# Личный кабинет таможенного брокера

Статический сайт без базы данных и backend. Все данные редактируются через
обычные `.json`-файлы: коды доступа, ФИО, баланс, ЭПТС, данные товара, трекинг
и статусы транспорта.

## Готовые версии

После сборки создаются два архива:

```txt
broker-with-timer.zip     # версия с таймером
broker-without-timer.zip  # версия без таймера
```

Внутри каждого архива лежит готовый сайт для хостинга:

```txt
index.html
epts/
tracking/
favicon.png
access.json
assets/
clients/
README.md
```

Папки `scripts/`, `package.json`, `.git/` и исходная папка проекта для установки
на хостинг не нужны. Они используются только для проверки и сборки.

## Страницы сайта

```txt
/           # главная страница: ЭПТС и трекинг вместе
/epts/      # отдельная страница проверки ЭПТС
/tracking/  # отдельная страница трекинга транспорта
```

На всех страницах используется один и тот же вход по коду доступа. В адресной
строке не отображаются `index.html` и расширение `.html`.

## Как установить на хостинг

1. Откройте панель управления хостингом.
2. Найдите папку сайта. Обычно она называется `public_html`, `www`, `htdocs` или
   `site`.
3. Распакуйте нужный архив: с таймером или без таймера.
4. Загрузите на хостинг именно содержимое архива, а не папку с архивом целиком.
5. В корне сайта должны лежать файлы:

```txt
index.html
epts/
tracking/
access.json
favicon.png
assets/
clients/
README.md
```

6. Откройте домен в браузере.
7. Введите тестовый код доступа, например:

```txt
987654
```

Если сайт открывается не на главной странице, проверьте, что `index.html` лежит
именно в корне сайта.

## Как установить на GitHub Pages

1. Создайте репозиторий на GitHub или откройте уже созданный.
2. Загрузите в репозиторий содержимое выбранного архива.
3. Откройте `Settings` -> `Pages`.
4. В блоке `Build and deployment` выберите:

```txt
Source: Deploy from a branch
Branch: main
Folder: / (root)
```

5. Нажмите `Save`.
6. Подождите 1-3 минуты, пока GitHub Pages соберет сайт.
7. Сайт будет доступен по ссылке вида:

```txt
https://USER.github.io/REPOSITORY/
```

Если после обновления файлы еще старые, сделайте жесткое обновление страницы:
`Ctrl + F5` на Windows или `Cmd + Shift + R` на macOS.

## Как устроены данные

Главный файл кодов:

```txt
access.json
```

Он связывает код доступа с файлом клиента.

Пример:

```json
{
  "987654": "clients/987654.json",
  "482915": "clients/482915.json"
}
```

Если пользователь вводит `987654`, сайт открывает файл:

```txt
clients/987654.json
```

Файлы клиентов лежат в папке:

```txt
clients/
```

Шаблон для нового клиента:

```txt
clients/_template.json
```

## Как добавить нового клиента

1. Откройте папку `clients`.
2. Скопируйте файл `_template.json`.
3. Переименуйте копию в новый код доступа, например:

```txt
555777.json
```

4. Откройте файл `555777.json` через Блокнот или любой текстовый редактор.
5. Измените ФИО, баланс, ЭПТС, транспорт, фото и статусы.
6. Откройте главный файл `access.json`.
7. Добавьте новый код:

```json
{
  "987654": "clients/987654.json",
  "482915": "clients/482915.json",
  "555777": "clients/555777.json"
}
```

Важно: между строками должны быть запятые, но после последней строки запятая не
ставится.

После этого пользователь сможет ввести код:

```txt
555777
```

и попасть в свой кабинет.

## Как менять ФИО, баланс, сальдо и остаток

Откройте файл клиента, например:

```txt
clients/987654.json
```

В начале файла есть блок `profile`:

```json
"profile": {
  "fullName": "Иванов Иван Алексеевич",
  "balance": "1 250 000 ₽",
  "saldo": "1 180 000 ₽",
  "remainder": "980 000 ₽",
  "notice": "Документы по текущей поставке приняты. Следующий этап — проверка ЭПТС и выпуск таможней."
}
```

Можно менять:

- `fullName` — ФИО клиента;
- `balance` — баланс счета;
- `saldo` — сальдо;
- `remainder` — остаток;
- `notice` — текстовый блок на главной странице.

## Как менять таймер

Таймер редактируется в файле клиента, например:

```txt
clients/987654.json
```

Найдите блок `timer`:

```json
"timer": {
  "enabled": true,
  "label": "Осталось времени:",
  "days": 1,
  "hours": 0,
  "minutes": 0,
  "seconds": 0
}
```

Что можно менять:

- `enabled` — включить или выключить таймер: `true` или `false`;
- `label` — подпись над таймером;
- `days` — количество дней;
- `hours` — количество часов;
- `minutes` — количество минут;
- `seconds` — количество секунд.

Пример на 2 дня, 5 часов и 30 минут:

```json
"timer": {
  "enabled": true,
  "label": "Осталось времени:",
  "days": 2,
  "hours": 5,
  "minutes": 30,
  "seconds": 0
}
```

Таймер стартует при входе в кабинет и идет до нуля. При переходе между
страницами `Главная`, `ЭПТС` и `Трекинг` он не сбрасывается. Если нажать
`Выход`, таймер очистится и при следующем входе начнется заново.

## Как менять ЭПТС и данные товара

В файле клиента найдите блок:

```json
"eptsRecords": {
```

Каждая запись внутри этого блока — отдельный номер ЭПТС.

Пример:

```json
"1643012345678901": {
  "epts": "1643012345678901",
  "lotPhoto": "assets/lot-car.jpg",
  "title": "Toyota RAV4 2.0, 2022",
  "brand": "Toyota RAV4",
  "year": "2022",
  "vin": "JTM863FV0ND123456",
  "body": "JTM863FV0ND123456",
  "color": "Белый",
  "type": "Легковой автомобиль",
  "engineVolume": "1987 см³",
  "power": "149 л.с.",
  "country": "Япония",
  "loadMass": "1 615 кг",
  "maxMass": "2 185 кг",
  "eco": "Евро-5",
  "engineType": "Бензиновый",
  "drive": "Полный",
  "gearbox": "Вариатор",
  "status": "Действителен",
  "statusDate": "01.04.2026"
}
```

Чтобы добавить еще один ЭПТС, скопируйте весь такой блок, вставьте ниже через
запятую и поменяйте номер/данные.

## Как менять трекинг транспорта

В файле клиента найдите блок:

```json
"trackingCodes": {
```

Пример трекинга:

```json
"BB-123456": {
  "transportName": "Автовоз Владивосток — Москва",
  "photo": "assets/tracking-truck.jpg",
  "statuses": [
    {
      "date": "01.04.2026",
      "text": "Автомобиль прибыл на оформление."
    },
    {
      "date": "03.04.2026",
      "text": "Подана декларация"
    }
  ]
}
```

Можно менять:

- `BB-123456` — код трекинга, который вводит пользователь;
- `transportName` — название транспорта;
- `photo` — фото транспорта;
- `statuses` — список статусов с датами.

Чтобы добавить новый статус, добавьте новый объект в массив `statuses`:

```json
{
  "date": "07.04.2026",
  "text": "Выпущен таможней"
}
```

Важно: между статусами должна быть запятая.

## Как менять фотографии

Все картинки лежат в папке:

```txt
assets/
```

Чтобы заменить фото:

1. Загрузите новую картинку в папку `assets`.
2. Желательно использовать латинские названия без пробелов, например:

```txt
lot-toyota.jpg
truck-ms123.jpg
```

3. В JSON укажите путь к картинке:

```json
"lotPhoto": "assets/lot-toyota.jpg"
```

или:

```json
"photo": "assets/truck-ms123.jpg"
```

Если фото лота нет, можно оставить пустую строку:

```json
"lotPhoto": ""
```

Тогда сайт покажет пустой блок/ошибку изображения, как было предусмотрено в
задаче.

## Как проверить, что JSON не сломан

JSON чувствителен к кавычкам и запятым.

Частые ошибки:

- удалили кавычку;
- поставили лишнюю запятую после последнего элемента;
- забыли запятую между строками;
- случайно удалили фигурную скобку `{` или `}`.

Если сайт после редактирования не открывает данные, сначала проверьте файл через
любой онлайн JSON validator или попросите разработчика запустить проверку:

```bash
npm run validate:data
```


## Важно про безопасность

Этот проект сделан без базы данных и backend, как было согласовано в задаче.
Такой вариант подходит для демо, простого кабинета.

Но JSON-файлы являются обычными файлами сайта. Это значит, что для настоящей
приватности клиентских данных нужен backend с авторизацией и хранением данных на
сервере.
