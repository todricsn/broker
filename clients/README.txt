Как редактировать данные без базы данных:

Главный файл:
access.json

В нем написано, какой код доступа открывает какой файл клиента:
"987654": "clients/987654.json"
"5662424": "clients/5662424.json"

Файлы клиентов:
clients/987654.json
clients/5662424.json

В файле клиента редактируются:
- ФИО
- баланс
- сальдо
- остаток
- текстовое сообщение
- таймер обратного отсчета
- ЭПТС
- фото лота
- транспорт
- статусы транспорта

Фото:
1. Картинки кладутся в папку assets.
2. В JSON указывается путь, например:
   "assets/lot-car.jpg"
3. Если фото лота нет, оставьте пустую строку:
   "lotPhoto": ""
   Тогда сайт покажет блок IMAGE ERROR.

Как добавить нового клиента:

1. Скопируйте файл _template.json.
2. Переименуйте копию в код доступа клиента, например 555777.json.
3. Откройте новый файл через Блокнот и измените ФИО, баланс, ЭПТС, транспорт, статусы и фото.
4. Откройте главный файл access.json.
5. Добавьте строку:
   "555777": "clients/555777.json"
6. Сохраните файлы и загрузите их на сайт.

Пользователь вводит код 555777, сайт смотрит в access.json,
находит clients/555777.json и открывает кабинет с этими данными.

Как менять таймер:
В файле клиента найдите блок timer.

Пример:
"timer": {
  "enabled": true,
  "label": "Осталось времени:",
  "days": 1,
  "hours": 0,
  "minutes": 0,
  "seconds": 0
}

days - дни
hours - часы
minutes - минуты
seconds - секунды

Таймер стартует при входе в кабинет и идет до нуля.
Если нажать Выход, при следующем входе таймер начнется заново.

Страницы:
- index.html — главная страница с ЭПТС и трекингом вместе.
- epts.html — отдельная страница проверки ЭПТС.
- tracking.html — отдельная страница трекинга транспорта.

Важно:
Не удаляйте запятые и кавычки в JSON.
Если после правки сайт не открывает данные, скорее всего, в файле пропущена запятая или кавычка.

Этот вариант работает без базы данных и backend. JSON-файлы являются обычными файлами сайта,
поэтому для реальных закрытых данных нужна серверная авторизация.
